# v0.1.0 Release Preparation

## Goal

Ship agentbox's first public GitHub Release. A first-time user should go from install to running without friction. Fix anything that would confuse or block them, and lock down the bridge feature.

## Scope

GitHub Release only. No crates.io. No Homebrew.

## 1. Code fixes

### 1.1 Show build errors on failure (image.rs)

When `verbose` is false and a build fails, we throw away captured stderr. The user gets "container build failed" and nothing else.

Fix: on the first-attempt failure path, always attach captured stderr to the error. `--verbose` still controls whether stderr streams live during the build, but when the build breaks, show the user why. (The buildkit-retry path already inherits stderr directly to the terminal, so it doesn't need this fix.)

### 1.2 Fix placeholder repo URL (image.rs)

`GIT_REPO_URL` still points to `https://github.com/AGuscable/agentbox.git`. Change it to `https://github.com/Rag0n/agentbox.git`.

### 1.3 Rename _bridge_handle (main.rs)

`_bridge_handle` appears on multiple lines and does real work, but the underscore prefix tells Rust "I'm unused." Rename it to `bridge_handle`.

### 1.4 Update .gitignore

Add `.DS_Store`.

## 2. Install script

Create `install.sh` at repo root (~40 lines):

1. Verify architecture is `arm64`; bail otherwise (Apple Silicon only)
2. Verify OS is `darwin`; bail otherwise
3. Set install directory to `~/.local/bin`, or `INSTALL_DIR` if the user set it
4. Create the install directory if it doesn't exist
5. Download `agentbox-darwin-arm64.tar.gz` from the latest GitHub Release
6. Extract the binary, move it to the install directory
7. Warn if the install directory is not in `$PATH`
8. Print success with version output

Usage: `curl -fsSL https://raw.githubusercontent.com/Rag0n/agentbox/main/install.sh | bash`

## 3. GitHub Actions release workflow

Create `.github/workflows/release.yml`. Trigger on tag push matching `v*`:

1. Run on `macos-15` (Apple Silicon). Pin the runner — don't use `macos-latest` since GitHub shifts the alias over time
2. Build with `cargo build --release`
3. Package the binary into `agentbox-darwin-arm64.tar.gz`
4. Create a GitHub Release named after the tag
5. Upload the tarball as a release asset

Requires `permissions: contents: write` so the workflow can create releases and upload assets.

No other CI for now. No tests, no linting, no multi-platform.

## 4. README rewrite

What changes from the current README:

- Fix URLs: swap the `<user>` placeholder for `Rag0n` in download links
- Lead with the install script: `curl | bash` becomes the primary method. Keep `curl | tar` as the manual fallback. Drop `cargo install` (we are not publishing to crates.io). Drop "Homebrew (coming soon)"
- Mark bridge as experimental: add "Host Command Execution (Experimental)" near the bottom. Describe what it does, how to configure `allowed_commands`, flag it as experimental
- Cut the fat: parts of the current README read like a design spec. Shorten sentences, kill redundancy
- Keep the existing section order: Requirements, Install, Quick Start, Configuration, Custom Dockerfiles, Authentication, What's Mounted, Sharing Screenshots, What's Isolated, How It Works

## 5. Release checklist

After merging everything to main:

0. Verify `Cargo.toml` version matches the intended tag
1. `cargo build --release && cargo test` -- clean build, passing tests
2. `git tag v0.1.0`
3. `git push origin v0.1.0` -- triggers CI
4. Confirm the GitHub Release appears with the binary attached
5. `curl -fsSL https://raw.githubusercontent.com/Rag0n/agentbox/main/install.sh | bash` -- confirm the script works end to end

## Out of scope

- crates.io publish
- Homebrew tap or formula
- PID reuse race in bridge cleanup (theoretical, resets each session)
- Binary data handling in hostexec (edge case; these are text-oriented commands)
- Memory config validation (bad values already fail at container runtime with readable errors)
- More CI (tests, linting)
- Multi-platform builds (macOS ARM only for now)
